ЧАСТЬ 1. Видео YouTube: конспект и транскрипт (авто-субтитры)
================================================================================

Ссылка: https://www.youtube.com/watch?v=d75OTGiyA1w
Название (по метаданным): Это САМЫЙ ДЕШЕВЫЙ и ПРОСТОЙ способ настроить OpenClaw (Clawdbot)
Длительность: ~7:41

Источник текста: официальные авто-субтитры YouTube (русская дорожка), очищены дубли и
разметка VTT. Возможны искажения имён (OpenCla, Gemai, boter и т.п.) — сверять с
описанием под видео и документацией OpenClaw.

--------------------------------------------------------------------------------
Краткий связный текст (по смыслу ролика)
--------------------------------------------------------------------------------

Сейчас практически весь рынок AI сходит с ума от сервиса под названием OpenClaw
(раньше назывался Clawdbot). Смысл в том, что из обычного Telegram-бота можно
сделать мультиагентную систему с доступом по умолчанию почти к любым подключаемым
сервисам; настраивается в пару кликов, в итоге бот может управлять многими вещами
и выполнять разные действия.

Многие покупают отдельные компьютеры вроде Mac mini, чтобы поставить сервис локально
— из соображений безопасности, чтобы не давать ему доступ ко всем файлам на основном
ПК. Автор предлагает дешевле: за несколько сотен рублей и 5–7 минут поставить всё
на арендованный VPS, плюс подключить Gemini с Pro-подпиской на год и общаться с
OpenClaw через него, потратив порядка 1000–1500 ₽.

VPS: регистрация на платформе хостинга (в оригинале звучит как «Bet» — вероятно,
конкретный провайдер), пополнение баланса парой сотен рублей, панель управления →
виртуальный сервер → регион Латвия или Казахстан → задать и сохранить пароль →
создать сервер, ждать 3–5 минут, пока статус сменится на «создан».

SSH: взять строку подключения из панели, открыть Терминал (Windows/Mac), ввести
«ssh » и вставить строку, Enter, на запрос доверия — yes, затем ввести пароль
(символы могут не отображаться — вставить и Enter).

Установка OpenClaw: зайти на сайт проекта (ссылка в описании видео), в разделе
Quickstart скопировать команду установки, вставить в терминал, Enter, подождать
2–3 минуты.

Настройка: согласиться с вопросами, Quick Start, выбрать провайдера LLM — автор
выбирает Google, чтобы использовать Gemini по подписке, а не через API-ключи
(API для него «дорого по токенам»). Выбрать Google Gemini CLI OAuth; если CLI не
установлен — выполнить показанную команду (npm / установка Gemini CLI). Затем
команда вроде openclaw onboard (точное написание — в инструкции автора), снова
пройти активацию, выбрать Google и Gemini CLI, открыть выданную ссылку в браузере,
войти в аккаунт с Gemini Pro, на «странице успеха» скопировать URL из адресной
строки, вставить в терминал, Enter.

Модель: Gemini 3 Pro или Gemini 3 Flash — автор для скорости берёт Flash, для
«более умной» модели советует Pro, лимитов хватит на обе.

Telegram: подключить мессенджер — автор выбирает Telegram, вставить токен бота.
Токен получить у BotFather в Telegram: /start → New Bot → имя и username →
скопировать токен в терминал. Дополнительные настройки можно пропустить.
Установится Gateway — нажать предложенную кнопку/действие в интерфейсе. Затем
привязать свой Telegram к боту: в чате появится фраза вроде «Wake up my friend»,
нужно скопировать последнюю фразу/код и дописать в терминал как показано, Enter.
После этого можно писать боту в Telegram; ответ может долго «печататься» — это
нормально.

Итог: за 5–7 минут настроен бот OpenClaw в Telegram; терминал можно закрыть,
дальше работа в мессенджере: инструкции, подключение внешних сервисов, CRM,
Google Таблицы, Notion и т.д. Обещано отдельное видео и файл с командами; ссылки
на дешёвые аккаунты Gemini Pro и инструкцию автор обещает в закрепе Telegram-канала.


================================================================================
ЧАСТЬ 2. Как подключать бота не по API, а через существующую подписку на AI
================================================================================

Имеется в виду сценарий из того же видео: не использовать API-ключ (поминутная /
потоковая оплата за токены через Google AI API и т.п.), а привязать аккаунт с
активной подпиской Gemini (Pro/Ultra и аналоги) через OAuth и Gemini CLI.

--------------------------------------------------------------------------------
Разница подходов
--------------------------------------------------------------------------------

API-ключ:
  Запросы идут в оплачиваемый по токенам API. Для агента с большим контекстом
  расход может быть высоким.

Подписка через OAuth + Gemini CLI:
  Вход как пользователь Google с активной подпиской на Gemini. Доступ через
  официальный Gemini CLI и OAuth, не через отдельный биллинг API-ключа.
  Используются лимиты и условия подписки, а не «чистая» оплата API за токены
  в том же виде.

--------------------------------------------------------------------------------
Что выбрать в мастере настройки OpenClaw
--------------------------------------------------------------------------------

1. Провайдер модели: Google (не путь «только API key» для этой цели).
2. Вариант: Google Gemini CLI с OAuth (в субтитрах видео — «Gemini CLI OA»).
3. Не выбирать настройку, где для Gemini задаётся только API-ключ, если цель —
   именно использование существующей подписки.

--------------------------------------------------------------------------------
Логика шагов (по смыслу видео)
--------------------------------------------------------------------------------

1. Установить Gemini CLI командой с сайта OpenClaw (в ролике — через npm на том же
   сервере, где крутится бот).

2. Запустить openclaw onboard (или эквивалент из Quickstart) и снова выбрать
   Google → Gemini CLI.

3. Открыть в браузере ссылку из терминала.

4. Войти в тот Google-аккаунт, на котором уже есть подписка Gemini (Pro и т.д.),
   подтвердить доступ.

5. После редиректа скопировать URL из адресной строки и вставить обратно в
   терминал — завершение OAuth и привязка «подписка → CLI → OpenClaw».

6. Выбор модели (Flash / Pro и т.д.) уже в рамках этого входа, без отдельного
   API-ключа.

--------------------------------------------------------------------------------
Замечания
--------------------------------------------------------------------------------

— Точные пункты меню и команды могут меняться в новых версиях OpenClaw; ориентир:
  Gemini CLI + OAuth, не API key.

— Файл OpenClaw.md в этом репозитории описывает другую архитектуру (свой SaaS-бот
  и API к Claude/OpenAI к self-hosted OpenClaw) — это не то же самое, что подписка
  Google на Gemini из видео.

— Актуальные шаги всегда сверять с официальным Quickstart на сайте OpenClaw.
